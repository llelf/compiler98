Installing nhc98 under Windows

System Requirements

In order to use nhc98 under Windows 95/98 or NT, you first need to install the Cygnus gcc C compiler and tools. Essentially, it should be as simple as downloading cygwin/full.exe from your favorite Gnu-mirroring FTP site, and running it (it is a self-extracting archive). We last tested with release b19, but others have reported success with b20.1. (In Northern Europe, a good mirror of the Gnu tools is SunSite.)

Some people have reported difficulty getting recent pre-releases of nhc98 to build with Cygwin. The make fails very quickly, saying that there is "no rule to make target .../src/runtime/Kernel". We aren't yet exactly sure of the cause. Here is one suggestion - make sure that the environment variable MAKE_MODE is set to UNIX. Another suggestion is that Gnu-Make's $(patsubst ...) can't handle spaces in directory pathnames - so please make sure you don't have any! If either of these fix the problem, please let me know so that I can count this as a dead bug rather than an annoyingly still-alive bug.

Along with gcc, you also get the bash shell and lots of other Unix tools, such as tar, gunzip, and make, which are needed to unpack and build the nhc98 distribution. For full information on the Gnu Cygnus toolset, see http://www.cygnus.com/misc/gnu-win32/

And now...

Once you have installed gcc and tools, the standard installation instructions apply. You have the option of installing the binary or source distribution packages.

The latest updates to these pages are available on the WWW from http://www.cs.york.ac.uk/fp/nhc98/
